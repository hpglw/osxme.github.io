越简单越难搞！记Lenovo ThinkPad X1 Carbon i5三代老 Laptop Hackintosh Mojave 10.14.6

客户一台Lenovo ThinkPad X1 Carbon Laptop ，应该是X1 CARBON 一代产品，CPU 是i5三代的3317U,没有独显， IGPU HD 4000,Ivy Bridge架构。
Motherboard Lenovo 3448aw5 QS77的，4GBMemory ，1600X900分辨率显示屏， ALC 269Audio ，没有自带 LAN Network ， WIFI Network 为6205无解。
180GB固态 Install 单系统，按这个配置来说，10.14.6 Mojave是最适合的。

下面讲讲 Install 时遇到的问题，根据配置，选用CLOVER引导，比较成熟，热补丁这些比较适合老机子，OC虽然新，但很多补丁需要手动，也就没有必要了。

HD 4000的 IGPU 非常好驱动，只需将ig-platform-id设置为0x01660004即可，其它都 用设置，因为没有独显，所以也 用制作屏蔽独显的SSDT。亮度调节丢SSDT-PNLF.aml进去即可。

ALC 269Audio 个人建议用VOODOOHDA万能驱动，比较快捷，当然APPLE ALC 也可以设置ID为6，但APPLE ALC 驱动有时会掉，看个人选择。

因为没有 LAN Network ， WIFI Network 也无法驱动，所以建议客户买个Hackintosh WIFI Network ，淘宝才10多块钱。

特别需要注意的是，Memory 修复补丁，如果出现重启后卡加号，建议升级成CLOVER R5127最新版本，用OpenRuntime.efi替代AptioMemoryFix.efi，这些Memory 补丁都太老旧了。

另外， 要用OC 6.3版，会出现OC引导界面卡死

Install 好之后， IGPU 驱动，亮度调节正常，Audio OK，触控板OK，电池电量有显示，基本完美。

MSI Z370 KRAIT GAMING**Motherboard** 搭配I7-8700K N卡GTX 1060Hackintosh 注意事项

配置图：

8代I7-8700K，MSI Z370 KRAIT GAMINGMotherboard ，32GB（两条16G 2400）,双NVME SSD ，一个970 EVO一个970 EVO PLUS，GTX 1060独显。

就以上的配置，Hackintosh MACOS 10.13.6版，看配置是相当简单，但往往越简单越容易出问题，以下总结问题所在。

首先CPU 没什么好说的，8代配置， IGPU 默认在BIOS是关闭的，开 开启问题 大，反正10.13.6也 UHD 630.

然后就是Motherboard MSI Z370 KRAIT GAMING，这个要留意三点：第一，BIOS必须更新到最新版本，尤其重要，我 Install 的时候这块Motherboard 仍然是2017年的BIOS版本，各种出错，需要先到官网下载最新的2020版本升级更新。第二，关闭CFG LOCK，尤其重要，这块Motherboard 能通过引导的补丁跳过CFG LOCK，需要在BIOS搜索CFG，将其禁用！第三，两块NVME SSD ，在 SSD 模式中，SATA的热拔插禁用，SATA模式一定要是ACHI。

然后Memory ，别小看这两条32GMemory ，用CLOVER的话，往往就会卡++++号，或者

卡在End Randdomseed 动，又或者couldn’t allocate runtime area错误，这些都是 能通过Memory 的错误，与其一个个去试Memory 补丁，还 如直接换OC引导。

再者就是 SSD 了，相当吓人的SAMSUNG SSD ，对Hackintosh 极度 友好的970 EVO PLUS版，记得先升级 SSD 固件到最新版本，怎么升级，百度一下就好了。

综合以上都解决之后，就可以一路畅通Hackintosh ，基本一步到位。

GIGABYTE P65 Laptop Hackintosh EFI Catalina 10.15.7

一台GIGABYTE P65 Laptop ，配置为i7-7700HQ,16GBMemory ，512G固态, IGPU HD 630，独显GTX 1060（需要屏蔽），1080P分辨率，INTEL WIFI Network 加REALTEK LAN Network 。

这次选用比较成熟的CLOVER引导，主要为了方便，其实OC差别 大。

首先BIOS设置：

1，确认SATA模式为AHCI

2，关闭安全启动
